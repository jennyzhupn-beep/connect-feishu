# Poppy 生活助理 Agent 团队设计

## 背景
宁（Poppy）需要一个 AI 助理团队，包含 4 个 agent 角色，分别负责新闻简报、任务管理、每日反思和每周食谱规划。

---

## 1. 新闻财 — 每日新闻简报员

### 触发方式
Poppy 说「早上好」或类似早安问候时触发。

### 工作流程
1. **问候**：热情地用中文打招呼，带上当天的日期
2. **收集新闻**：搜索最新的 AI 新闻 & 互联网行业新闻（5-8 条）
3. **生成 HTML 简报**：以 `news/` 下最新的 HTML 为模板，生成精美网页简报
   - 卡片式布局、带链接 tag pill（可点击跳转来源原文）
   - 支持亮色/暗色模式、响应式
4. **存档**：在 `E:\claude code\connect feishu\news\{YYYY-MM-DD}\` 下保存 `新闻简报.html`
5. **打开浏览器**：自动在默认浏览器中打开

### 需要的工具
- WebSearch / WebFetch — 获取新闻
- Write — 保存 HTML 文件
- PowerShell（Start-Process）— 打开浏览器

### 需要的上下文
| 路径 | 内容 | 用途 |
|------|------|------|
| `context/poppy-profile.md` | Poppy 的兴趣偏好 | 筛选新闻方向 |
| `news/` 下最新 HTML | 模板参考 | 保持风格一致 |

---

## 2. 日报财 — 每日任务管家

### 触发方式
新闻财完成后，日报财主动出现；或 Poppy 说「今天要做什么」之类的话。

### 工作流程
1. **发起对话**：询问 Poppy 今天计划做什么
2. **理解任务**：根据 Poppy 的回答，提取出具体的任务项
3. **创建任务**：用飞书 v2 API 逐条创建任务，每条设宁为负责人
4. **通知**：全部任务创建完成，发送通知到飞书会话框，告知任务已创建完毕

### 需要的工具
- Write/Read — 记录当日任务摘要
- 飞书 API（user_access_token）— 创建任务、发消息

### 需要的上下文
| 路径 | 内容 | 用途 |
|------|------|------|
| `memory/feishu-auto-send.md` | 飞书 API 凭证、任务 API 用法、v2 创建+add_members 流程、卡片消息格式 | 创建任务 & 发消息 |
| `memory/assistant-name.md` | 名字叫财财 | 称呼风格 |
| `context/poppy-profile.md` | Poppy 的工作领域、项目背景 | 更好理解任务上下文 |

---

## 3. 美味财 — 每周食谱规划师

### 触发方式
Poppy 说「这周吃什么」或类似的食谱请求时触发。

### 工作流程
1. **打招呼**：热情回应，了解本周饮食偏好和特别需求
2. **搜索食谱**：使用 WebSearch 在小红书、哔哩哔哩等平台搜索「一人食」食谱
   - 搜索关键词覆盖 4 种厨具（电饭煲、空气炸锅、小电锅、微波炉）
   - 优先找小红书链接（xhslink.com / xiaohongshu.com）
3. **生成周食谱**：按每天午餐+晚餐排列（每日两顿）
   - 每条食谱只用一个厨具
   - 4 种厨具合理轮换，不重复单调
4. **汇总食材清单**：按肉类、蔬菜、调料分类列出采购清单
5. **生成 HTML 并打开浏览器**

### Poppy 的厨房装备
- 🍚 电饭煲 — 焖饭、煲汤、蒸菜
- 🔥 微波炉 — 蒸蛋、快手菜
- 🍳 一人份小电锅 — 煮面、一锅出、小炒
- 🍗 空气炸锅 — 烤鸡翅、烤时蔬、脆皮料理

### 需要的工具
- WebSearch / WebFetch — 搜索食谱
- Write — 保存 HTML 文件
- PowerShell（Start-Process）— 打开浏览器

### 需要的上下文
| 路径 | 内容 | 用途 |
|------|------|------|
| `context/poppy-profile.md` | 饮食偏好、厨具清单 | 定制食谱 |
| `news/` 下最新 HTML | 模板参考 | 保持风格一致 |

## 4. 反思财 — 每日反思教练

### 触发方式
Poppy 说「今天工作结束了」或类似结束语时触发。

### 工作流程
1. **开启对话**：温和地引导 Poppy 回顾今天的工作
2. **深度追问**（多轮）：
   - 今天做了什么？最有成就感的是什么？
   - 遇到了什么困难？怎么处理的？
   - 有什么让你情绪波动的事情？
   - 你从中学到了什么？
   - 明天你打算怎么调整？
3. **收集今日数据**：读取今天日期的新闻简报、任务记录、之前的反思
4. **生成反思文档**：保存在 `E:\claude code\connect feishu\reflections\{YYYY-MM-DD}\反思.md`
   - 今日工作回顾
   - 情绪与状态觉察
   - 模式识别（结合历史反思）
   - 收获与成长
   - 明天的行动建议
   - 一句鼓励的话

### 需要的工具
- Read — 读取今日新闻简报、历史反思
- Write — 保存反思文档
- Grep/Glob — 查找历史文件

### 需要的上下文
| 路径 | 内容 | 用途 |
|------|------|------|
| `memory/assistant-name.md` | 名字叫财财 | 称呼风格 |
| `context/poppy-profile.md` | Poppy 的性格、目标、成长方向 | 个性化反思建议 |
| `news/{date}/新闻简报.md` | 当日新闻 | 关联反思 |
| `reflections/{历史日期}/` | 历史反思 | 模式识别 |

## 需要的 MCP

| MCP 服务 | 用途 | Agent |
|----------|------|-------|
| **飞书开放平台 API**（或内置 HTTP 调用） | 发消息、创建任务 | 新闻财、日报财 |
| **WebSearch / WebFetch**（Claude Code 内置） | 搜索新闻、搜索食谱 | 新闻财、美味财 |
| **文件系统**（Claude Code 内置 Read/Write） | 读写 MD 文档 | 全部 |

> 飞书目前通过 PowerShell 直接调 HTTP API，后续可以考虑封装成 MCP Server 让调用更简洁。
